The Goal

Claim as many squares as you can.

  Rules

The game is played on a 7 by 7 board. Each player starts with 2 pieces.

In each turn a player can make the following moves to one of his pieces:
  • JUMP: A piece can be moved to one of the four adjacent empty squares(Up, Down, Right, Left), leaving the initial square empty.
  • REPL: A piece can be replicated to one of the four diagonal squares(Top-right, Top-left, Bottom-right, Bottom-left).

Moving or replicating a piece to one of the four adjacent squares of opponent's piece will convert it.
Player's score is calculated based on the number of pieces on the board. The game will end when:
  • All of the cells of the 7 by 7 board is filled.
  • One of the player runs out of moves. The empty squares will be filled with pieces of the other player.
  • After 200 moves the game will be forcefully ended. Player with the most pieces will be declared as winner.

  Game Input

Initilization Input

Line 1: one integer playerIndex - PlayerIndex 1 means opponent's index will 2 and vice versa.

For Each Game Loop

Line 1: one integer cellCount - Number of cells in the board.

Next cellCount lines: three integers row col state - Position and state of a cell in the board.
  • State -1: Cell is blocked.
  • State 0: Cell is empty.
  • State 1: Cell has a piece of player with index 1.
  • State 2: Cell has a piece of player with index 2.
Output
A single line containing player's move. Example: 5 1 REPL 6 2.
  • r1 c1 MOVE r2 c2: player's piece at row r1 and column c1 moves to row r2 and column c2. Cell (r1, c1) becomes empty.
  • r1 c1 REPL r2 c2: player's piece at row r1 and column c1 replicates to row r2 and column c2.
You must provide a valid command for exactly one piece each turn.
Constraints
playerIndex is either 1 or 2.
cellCount is 49.
0≤row≤6
0≤col≤6
-1≤state≤2
Response time per turn ≤ 100ms
